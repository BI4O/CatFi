# 一、产品概述

### 1.1 一句话描述

**一款通过养成 AI 猫咪来学习 DeFi 投资的游戏化应用**

### 1.2 核心理念

- 非目的性交易：通过游戏和宠物形象来吸引用户，而不是开门见山、枯燥的交易
- 面向普通人的增量市场：用通俗易懂的小猫养成系体系（零花钱、银行、种田）来简化所有的复杂概念
- AI内核：把AI物化为受用户习惯影响的宠物
- 低负担：前期可以用虚拟的宠物和虚拟的代币来模拟交易和金融

### 1.3 产品定位

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   传统交易所          CatFi            纯游戏             │
```

### 5.2 投资性格（动态变化）

猫的性格并非固定值，而是由用户的 **“标记行为”** 与 **“聊天内容”** 实时塑造的 **4D 意图模型**。

#### 1. 风险偏好 (Risk Appetite)

> **保守** `[ 🟢 ──── ⚪ ──── ⚪ ]` **激进**
>
> - ⬆️ **倾向保守**：多标记主流币（BTC/ETH）新闻为“关注”；标记山寨币为“存疑”。
> - ⬇️ **倾向激进**：频繁关注高波动、小市值项目的异动新闻。

#### 2. 决策速度 (Execution Speed)

> **谨慎** `[ ⚪ ──── ⚪ ──── 🟡 ]` **冲动**
>
> - ⬆️ **倾向谨慎**：对话中教育猫：“多看几天”、“再观察下”、“不急于一时”。
> - ⬇️ **倾向冲动**：对话中教育猫：“机会稍纵即逝”、“快买”、“速度跟上”。

#### 3. 信息辨别 (Information Insight)

> **易骗** `[ ⚪ ──── 🔵 ──── ⚪ ]` **怀疑**
>
> - ⬆️ **倾向怀疑**：能准确将误导性信息、假新闻标记为“存疑”。
> - ⬇️ **倾向易骗**：经常将营销号或虚假利好消息标记为“关注”。

#### 4. 持仓耐心 (Holding Patience)

> **短线** `[ ⚪ ──── ⚪ ──── 🟣 ]` **长线**
>
> - ⬆️ **倾向长线**：对话中教育猫：“拿住别卖”、“做时间的朋友”、“忽略短期波动”。
> - ⬇️ **倾向短线**：对话中教育猫：“见好就收”、“赚了就跑”、“落袋为安”。

# 八、技术需求

### 8.1 第三方服务

| 服务类型 | 用途                 | 建议方案                     |
| -------- | -------------------- | ---------------------------- |
| 新闻API  | 获取每日Crypto新闻   | CryptoPanic / CoinGecko News |
| 价格API  | 实时币价             | CoinGecko / CoinMarketCap    |

| 链上交互 | 真实交易（高级功能） | Ethers.js / Web3.js          |
| 钱包连接 | 导入钱包             | WalletConnect                |

### 8.3 AI 猫决策逻辑

```
每日结算流程：

1. 获取用户昨日标记的新闻
   ↓
2. 分析猫的当前性格参数
   ↓
3. 结合新闻内容 + 性格 → AI 决策
   ↓
4. 决定是否购买、购买什么、购买多少
   ↓
5. 执行模拟交易
   ↓
6. 生成日报内容
   ↓
7. 用户打开APP时展示


AI Prompt 示例：

你是一只名叫{cat_name}的小猫，正在学习投资。

你的性格参数：
- 风险偏好：{risk_score}/10
- 决策速度：{speed_score}/10
- 信息辨别：{trust_score}/10
- 持仓耐心：{patience_score}/10

你偏好的币种：{preferred_coins}
你不喜欢的币种：{disliked_coins}

今天主人标记为"关注"的新闻：
{liked_news_list}

今天主人标记为"存疑"的新闻：
{disliked_news_list}

你的零花钱有 {pocket_money} CAT。

请决定今天要不要买币，买什么，买多少。
用JSON格式返回。
```
